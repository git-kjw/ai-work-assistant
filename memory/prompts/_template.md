# {작업유형} 가이드 (Reference Only)

> 이 문서는 {작업유형} 작업을 수행할 때 참고하는 **기본 원칙과 권장 사항**입니다.
> **에이전트의 자율적 판단을 최우선**으로 하며, 프로젝트의 규모나 상황에 따라 자유롭게 범위를 조절합니다.

---

## 1. 개요 및 역할
이 가이드는 {작업유형} 시 발생할 수 있는 일반적인 상황에 대한 가이드라인을 제공합니다.

## 2. 권장 체크리스트 (참고용)
- [ ] 핵심 로직 파악
- [ ] 의존성 영향도 확인
- [ ] 관련 테스트 케이스 검토
*※ 위 리스트는 예시이며, 작업의 성격에 따라 생략하거나 확장할 수 있습니다.*

## 3. 상황별 판단 및 전략 (Adaptive Strategy)
에이전트는 다음 요소를 고려하여 작업의 깊이와 범위를 스스로 결정합니다:
- **프로젝트 규모**: 소규모 프로젝트는 신속하게, 대규모 프로젝트는 단계적으로 분석.
- **요청의 긴급도**: 단순 확인은 최소한으로, 구조적 변경은 심층적으로 접근.
- **도메인 복잡성**: Kafka, Redis 등 복잡한 미들웨어가 포함된 경우 해당 흐름을 중점 분석.

## 4. 지식 축적 규칙
- 분석 결과나 작업 과정에서 얻은 인사이트는 `memory/` 하위 적절한 폴더에 자동 기록하여 지식을 자산화합니다.
