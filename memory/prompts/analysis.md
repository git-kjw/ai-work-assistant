# 분석 가이드 (Reference Only)

> 이 문서는 코드 분석 작업을 수행할 때 참고하는 기본 원칙과 권장 사항입니다.
> 에이전트의 자율적 판단을 우선하며, 프로젝트 규모와 요청 목적에 따라 분석 범위를 조정합니다.

---

## 1. 개요 및 역할
요청된 프로젝트/모듈의 구조, 핵심 흐름, 위험 요소를 빠르게 파악하고 재사용 가능한 지식으로 정리한다.

## 2. 권장 체크리스트 (참고용)
- [ ] 진입점(앱/컨트롤러) 확인
- [ ] 핵심 상태/데이터 모델 확인
- [ ] 저장소/외부 연동 흐름 확인
- [ ] 주요 비즈니스 규칙 정리
- [ ] 잠재 리스크/미구현 영역 식별

## 3. 상황별 판단 및 전략 (Adaptive Strategy)
- **소규모 프로젝트**: 핵심 파일 중심으로 빠르게 End-to-End 흐름 정리
- **중대형 프로젝트**: 모듈별로 분리해 데이터/호출 흐름을 단계적으로 분석
- **변경 요청이 예정된 분석**: 영향도와 결합도를 우선 기록

## 4. 지식 축적 규칙
분석 결과는 최소 `projects/`(프로젝트 카드)와 `domains/`(기술 맵)에 반영하고, 후속 액션이 있으면 `tasks/`에 연결한다.
